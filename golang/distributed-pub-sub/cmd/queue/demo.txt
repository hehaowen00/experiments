# Message Queue Demo
# Run each command in a separate terminal.
# Demonstrates: overflow buffering, dead letter queue, delivery retries, disk-backed queues.

# --- Step 1: Start a producer ---

# Publishes a job message every second
go run ./cmd/queue/producer -grpc :19100 -advertise 127.0.0.1:19100 -source producer-1 -topic jobs -rate 1s

# --- Step 2: Start a consumer ---

# Processes jobs reliably (0% failure)
go run ./cmd/queue/consumer -grpc :19200 -advertise 127.0.0.1:19200 -seeds 127.0.0.1:19100 -id consumer-1 -topic jobs

# --- Step 3: Test dead letter queue ---

# Consumer with 30% simulated failure rate — failed jobs go to DLQ after retries
go run ./cmd/queue/consumer -grpc :19201 -advertise 127.0.0.1:19201 -seeds 127.0.0.1:19100 -id consumer-2 -topic jobs -fail-rate 0.3

# Watch for [DLQ] lines in the consumer output — these are messages that
# exhausted all retries and were republished to _dlq.jobs

# --- Step 4: Test overflow buffering ---

# Fast producer (10 msgs/sec) + slow consumer (buffer size 16)
# Messages that don't fit in the channel spill to the overflow queue
go run ./cmd/queue/producer -grpc :19102 -advertise 127.0.0.1:19102 -seeds 127.0.0.1:19100 -source fast-producer -topic jobs -rate 100ms

# The consumer stats (printed every 10s) will show the "overflowed" counter
# increasing. All messages are still delivered — none dropped.

# --- Step 5: Test disk-backed queue ---

# Consumer with file-backed overflow — survives restarts
go run ./cmd/queue/consumer -grpc :19202 -advertise 127.0.0.1:19202 -seeds 127.0.0.1:19100 -id disk-consumer -topic jobs -queue-dir /tmp/queue-demo

# 1. Let it process some messages
# 2. Stop it (Ctrl-C)
# 3. Start the producer again so messages accumulate
# 4. Restart the consumer — it picks up overflow from disk

# --- Step 6: Multi-node mesh ---

# Second producer on a different node
go run ./cmd/queue/producer -grpc :19103 -advertise 127.0.0.1:19103 -seeds 127.0.0.1:19100 -source producer-2 -topic jobs -rate 2s

# Messages from both producers are delivered to all consumers across the mesh.
# Each consumer sees every message (fan-out, not competing consumers).
